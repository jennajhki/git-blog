온체인 데이터란?

📅 2025년 6월 5일

1. On-Chain Data 온체인 데이터란?

블록체인 네트워크에 기록된 모든 데이터를 의미합니다. 탈중앙화된 환경에서 검증된 정보로 구성되며, 누구나 접근 가능하고 위·변조가 불가능한 특징을 지닙니다.

이 데이터는 스마트 컨트랙트 실행 정보, 트랜잭션 기록, 계정 상태 등 블록체인 상의 모든 활동을 포함하며, 블록체인의 투명성과 신뢰성을 높이는 데 핵심 역할을 합니다.

2. 온체인 데이터 구성 요소

(1) 트랜잭션 데이터

  • 트랜잭션 해시: 거래를 식별하는 고유한 값
  • 보낸 주소 (From): 송신자 계정
  • 받는 주소 (To): 수신자 또는 컨트랙트
  • 전송 금액: 이동한 암호화폐 또는 토큰
  • 가스 사용량: 처리에 소모된 연료
{
  "hash": "0xabc123...",
  "from": "0x1234...",
  "to": "0xabcd...",
  "value": "1 ETH",
  "gasUsed": 21000
}

(2) 블록 데이터

  • 블록 해시: 블록을 식별하는 해시
  • 블록 번호: 생성 순서를 나타냄
  • 타임스탬프: 생성 시간
  • 채굴자: 블록 생성자
  • 트랜잭션 목록: 포함된 거래 리스트
{
  "number": 15000000,
  "hash": "0xblockhash123...",
  "miner": "0xminerAddress...",
  "timestamp": 1678900000,
  "transactions": ["0xabc123...", "0xdef456..."]
}

(3) 상태 데이터

  • 계정 잔액: 특정 주소의 보유 자산
  • 컨트랙트 저장소: 스마트 컨트랙트 상태 변수
  • 토큰 보유량: ERC-20 등 토큰 잔액
{
  "address": "0xuser123...",
  "balance": "2.5 ETH"
}

3. 온체인 데이터 활용

사용 사례 설명
트랜잭션 모니터링 지갑/컨트랙트 트랜잭션 실시간 추적
디파이 분석 유동성, 대출, 스왑 데이터 확인
NFT 조회 소유자 및 거래 내역 분석
스마트 컨트랙트 상태 확인 컨트랙트 내부 변수 값 확인

4. 온체인 데이터를 가져오는 방법

  • 직접 노드 실행: Geth, Erigon, Besu 등 사용
  • RPC 인프라 서비스: Infura, Alchemy, QuickNode 등
  • Web3 SDK 활용: Web3.js, Ethers.js 등

1. 노드(Node)란?

블록체인은 블록과 트랜잭션 데이터를 관리하는 노드(Node)라는 소프트웨어가 실행되는 컴퓨터의 분산 네트워크입니다. 이러한 노드를 운영하기 위해서는 클라이언트(Client)라는 응용 프로그램이 필요합니다.

  • 예시: Bitcoin Core, Geth 등

하지만 클라이언트를 직접 실행하려면 많은 리소스가 필요합니다. 그래서 일반 사용자는 다른 노드와 통신하여 데이터를 받아오는 방식을 활용할 수 있습니다.

2. 노드 간 통신 - JSON-RPC

RPC (Remote Procedure Call)

RPC는 별도의 리모트 제어를 위한 복잡한 코딩 없이 다른 네트워크 주소에 존재하는 함수나 프로시저를 실행할 수 있게 해주는 통신 방식입니다.

  • 분산 컴퓨팅 환경에서 많이 사용
  • MSA(마이크로 서비스 아키텍처) 구조에서 서비스 간 통신 수단으로 활용
  • 언어 및 플랫폼과 무관하게 서비스 호출 가능

Procedure 프로시저 란? 일련의 로직을 절차적으로 기술한 함수를 의미합니다.

JSON-RPC란?

JSON-RPC는 무상태성(Stateless)의 경량 RPC 프로토콜로, JSON 포맷으로 데이터를 주고받습니다. 다양한 메시지 전달 환경에서 사용할 수 있도록 HTTP, WebSocket 등과 추상화되어 있으며, 블록체인 노드와의 통신에 최적화되어 있습니다.

JSON-RPC의 예시 구조

{
  "jsonrpc": "2.0",
  "method": "eth_blockNumber",
  "params": [],
  "id": 1
}

위 예시는 최신 블록 번호를 조회하는 JSON-RPC 요청입니다.

3. 개발자용 라이브러리

JSON-RPC는 직관적인 구조지만, 개발자들이 더 쉽게 접근할 수 있도록 다양한 라이브러리가 존재합니다.

  • Web3.js: 이더리움과 상호작용할 수 있는 대표적인 자바스크립트 라이브러리
  • Ethers.js: 경량화되고 더 안전한 접근성을 제공하는 라이브러리

이런 라이브러리 덕분에 JSON-RPC의 원리를 몰라도 dApp 개발자는 한 줄 코드로 노드와 통신할 수 있습니다.
