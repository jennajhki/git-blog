REST API 이해하기

📅 2025년 5월 29일

오늘은 REST API의 개념과 RESTful한 방식으로 API를 설계하는 방법에 대해 배웠다.

1. REST란?

REST(Representational State Transfer)는 HTTP를 기반으로 리소스(데이터)를 URI로 표현하고, 요청/응답을 명확한 규칙에 따라 주고받는 아키텍처 스타일이다.

2. REST API의 기본 구성

  • 자원(Resource): URI로 표현됨 (예: /posts/1)
  • 행위(Verb): HTTP 메서드 (GET, POST, PUT, DELETE)
  • 표현(Representation): JSON, XML 등으로 응답

3. REST 성숙도 모델

📍 0단계

HTTP만 사용. RESTful하지 않은 초기 단계

📍 1단계

각 리소스마다 개별 URI를 부여함. 명사형으로 작성하며, 동사나 상태를 URI에 쓰지 않음.

📍 2단계

HTTP 메서드를 CRUD에 맞게 사용함:

  • GET: 조회
  • POST: 생성
  • PUT: 전체 교체
  • PATCH: 부분 수정
  • DELETE: 삭제

멱등성: 같은 요청을 여러 번 보내도 결과가 같아야 함 (GET, PUT, DELETE는 멱등, POST는 아님)

📍 3단계 (HATEOAS)

응답에 다음에 할 수 있는 행동을 안내하는 링크를 포함하여 하이퍼미디어 컨트롤을 제공함

4. Open API와 API Key

공공 데이터 포털(OpenAPI)이나 OpenWeatherMap과 같은 사이트는 누구나 접근 가능한 API를 제공한다.

API Key는 인증 수단으로, 서버 접근을 제한하고 사용량을 관리하는 데 사용된다.

Postman 시작하기

웹 개발에서 사용하는 대표적인 클라이언트는 브라우저입니다. 하지만 POST, PUT, DELETE 같은 요청은 Postman을 사용하여 보다 쉽게 테스트할 수 있습니다.

Postman이란?

  • HTTP 요청을 시각적으로 테스트할 수 있는 도구
  • GET, POST, PUT, DELETE 등 다양한 메서드 지원
  • 요청 헤더, 바디, 파라미터 설정 가능

사용 절차

  1. 공식 사이트에서 설치: Postman.com
  2. 회원가입 및 로그인
  3. New → HTTP Request 선택
  4. 요청 URL 및 메서드 입력
  5. 헤더/바디 작성 후 Send 클릭

오늘의 정리

  • REST는 단순한 HTTP API와 구별되는 명확한 설계 원칙이 있다
  • REST 성숙도 모델을 통해 API를 점진적으로 RESTful하게 설계할 수 있다
  • Open API를 통해 외부 데이터에 쉽게 접근할 수 있으며, API Key는 보안을 위한 필수 도구이다
  • Postman은 API 테스트에 매우 유용한 도구다
