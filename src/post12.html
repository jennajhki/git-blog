브라우저 네트워크 이해하기!

📅 2024년 5월 28일

오늘은 정말 어려웠던 하루였다. 브라우저의 네트워크 구조, HTTP/HTTPS, AJAX, 그리고 SSR과 CSR 개념까지 배웠는데, 모든 것이 새롭고 복잡했다. 그럼에도 불구하고 이 내용을 정리해보면서 머릿속을 정리해보려 한다.

브라우저 네트워크 구조

브라우저는 사용자에게 보이는 화면을 구성하는 동시에, 눈에 보이지 않는 여러 네트워크 요청을 통해 서버와 통신한다. 이때 사용하는 대표적인 프로토콜이 바로 HTTP이다.

HTTP란?

  • HTTP (HyperText Transfer Protocol): 클라이언트와 서버 간의 통신을 위한 프로토콜
  • 웹 페이지 로딩: HTML, CSS, JS 등의 리소스를 요청
  • API 통신: JSON 데이터 송수신
  • Stateless: 이전 요청 상태를 저장하지 않음

HTTPS

  • 기밀성: 데이터 암호화
  • 무결성: 데이터 변조 방지
  • 인증: 신뢰성 있는 서버인지 확인

HTTP 메시지 구조

요청(Request)

  • Start Line: 예) GET /index.html HTTP/1.1
  • Headers: 요청 정보 포함
  • Body: POST 방식 등에서 데이터 포함

응답(Response)

  • Status Line: 예) HTTP/1.1 200 OK
  • Headers: 응답 정보 포함
  • Body: HTML, JSON 등

2-Tier Architecture (클라이언트-서버 구조)

  • 클라이언트: 리소스를 사용하는 앱
  • 서버: 리소스를 제공하는 시스템

URI / URL / DNS

  • URL: 리소스의 주소
  • URI: URL + query, fragment 등 포함 식별자
  • DNS: 도메인을 IP 주소로 변환

AJAX

비동기적으로 서버와 통신할 수 있게 해주는 기술.

  • Fetch API로 요청
  • 페이지 전체 새로고침 없이 일부만 갱신
  • 장점: 빠른 응답, 유저 경험 향상
  • 단점: SEO 불리, History 문제

SSR vs CSR

SSR (Server-Side Rendering)

  • 서버에서 HTML 생성 후 클라이언트 전달
  • 초기 로딩 빠름
  • SEO에 유리

CSR (Client-Side Rendering)

  • HTML 뼈대 + JS → 클라이언트에서 렌더링
  • SPA에서 유리
  • 서버 부담 적음, 빠른 라우팅
