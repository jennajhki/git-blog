TypeScript

📅 2025년 5월 31일

오늘은 JavaScript의 상위 집합 언어인 TypeScript에 대해 배웠다. 타입 안정성과 유지 보수성을 높여주는 언어로, 다양한 타입 시스템을 배워보았다.

TypeScript 등장

JavaScript의 동적 타입은 유연하지만, 대규모 프로젝트에서 유지보수가 어렵다는 단점이 있다. 이를 보완하기 위해 정적 타입 검사 기능을 도입한 TypeScript가 등장했다.

기본 타입

Boolean

let isShow: boolean = true;

Number

let num: number = 42;
let pi: number = 3.14;

String

let name: string = "Alice";

Array

let fruits: string[] = ["apple", "banana"];
let nums: Array<number> = [1, 2, 3];

Tuple

let user: [string, number] = ["Alice", 30];

Object

let user: { name: string; age: number } = {
  name: "Bob",
  age: 25
};

Any

let value: any = 10;
value = "hello";

함수

function add(x: number, y: number): number {
  return x + y;
}

기본값과 선택 매개변수

function greet(name: string = "guest") {
  console.log(`Hello, ${name}`);
}

greet(); // Hello, guest

유니온 타입

function print(value: number | string) {
  console.log(value);
}

타입 가드

function print(value: number | string) {
  if (typeof value === "string") {
    console.log(value.toUpperCase());
  }
}

인터섹션 타입

interface Person {
  name: string;
}

interface Employee {
  employeeId: number;
}

let worker: Person & Employee = {
  name: "Jane",
  employeeId: 123
};

TypeScript 열거형(Enum), 인터페이스(Interface), 타입 별칭(Type Alias)

열거형 (Enum)

TypeScript에서 enum은 특정 값의 집합을 정의할 때 사용됨. JavaScript에는 존재하지 않지만 TypeScript는 이를 지원함.

숫자형 열거형

enum Color {
Red = 1,
Green = 2,
Blue = 4
}

값은 자동 증가하거나 명시적으로 정의, 산술 연산도 가능

문자형 열거형

enum Direction {
Up = "UP",
Down = "DOWN",
Left = "LEFT",
Right = "RIGHT"
}

디버깅이 명확하고, 주로 외부 API 값과 연계 시 유용

역 매핑

enum Enum {
A
}
let a = Enum.A;
let nameOfA = Enum[a]; // "A"

숫자형 열거형에서만 사용 가능하며, 값으로 키를 얻을 수 있음

인터페이스 (Interface)

interface는 객체의 구조를 정의하여 일관성 유지

변수와 인터페이스

interface User {
name: string;
age?: number; // 선택적 프로퍼티
}

선택적 프로퍼티는 ? 연산자 사용

함수와 인터페이스

interface Greeting {
(user: User, greeting: string): string;
}

클래스와 인터페이스

interface Calculator {
add(x: number, y: number): number;
substract(x: number, y: number): number;
}

implements 키워드로 인터페이스 강제 구현 가능

인터페이스 확장

interface Person {
name: string;
age: number;
}
interface Developer extends Person {
language: string;
}

타입 별칭 (Type Alias)

type MyString = string;

타입을 간단히 표현하고 재사용성 높임

타입 별칭과 인터페이스 비교

// 에러 발생
// type Students extends Person { className: string; }

// 가능
interface Students extends Person {
className: string;
}

타입 별칭은 확장 불가, 인터페이스는 확장 가능하므로 확장성이 필요하면 인터페이스 사용 권장

타입 추론 (Type Inference)

타입스크립트는 변수 초기값을 기반으로 타입 자동 추론

let isNumber = 123; // number로 추론

최적 공통 타입

let x = [0, 1, null];

배열의 타입을 종합해 공통 타입 추론

문맥상 타이핑

사용되는 문맥에 따라 타입이 추론되기도 함

클래스 (Class)

접근 제어자

  • public: 외부에서 접근 가능
  • private: 외부 접근 불가
  • readonly: 읽기 전용
class Mydog {
readonly name: string;
constructor(theName: string) {
  this.name = theName;
}
}

제네릭 (Generic)

function printLog<T>(text: T): T {
return text;
}

재사용 가능한 함수, 인터페이스, 클래스 작성에 유용

제약 조건 (Constraints)

interface TextLength {
length: number;
}
function printLog<T extends TextLength>(text: T): T {
console.log(text.length);
return text;
}

keyof를 활용한 제약 가능:

function printLog<T extends keyof Item>(text: T): T {
return text;
}

제네릭을 잘 사용하면 타입 안정성과 코드 재사용성을 동시에 확보할 수 있음
